# siteofficial-i-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定域名或网站，仅作为静态资源存储与展示的通用集合。

项目主要用于保存历史页面快照、实验性前端页面、临时展示页等，方便快速访问和版本管理。

## 目录说明

```
siteofficial-i-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（如图片、样式、脚本）
├── docs/           # 可选：辅助文档或说明文件
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件独立存放，文件名建议使用有意义的英文或数字标识。
- `assets/`：按需组织，可包含子目录，建议与对应页面保持一致的命名约定。

## 页面归档说明

- 所有页面均为纯前端静态页面，不依赖后端服务。
- 每个页面应自包含或明确引用 `assets/` 下的资源。
- 归档时请尽量保持页面原始结构，如需修改请注明变更记录。

## 维护说明

- 本仓库由维护者不定期更新，欢迎提交 Pull Request 添加新页面或改进现有内容。
- 提交前请确保页面无恶意代码、无外部跟踪脚本，并遵守相关法律法规。
- 对于过期或不再需要的页面，建议保留历史记录，避免直接删除。

## 贡献指南

1. Fork 本仓库。
2. 在 `pages/` 下添加新 HTML 文件，或在 `assets/` 中添加资源。
3. 提交 Pull Request 并简要描述变更内容。
4. 维护者审核后合并。
